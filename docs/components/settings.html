Architecture

Window chrome

SettingsWindow owns the Adw.NavigationSplitView, sidebar Gtk.ListBox, content Gtk.Stack, search bar, and Adw.ToastOverlay.

Lazy pages

_build_pages instantiates page objects cheaply; _ensure_page_built calls build() only on first selection and caches the widget in the stack.

Responsive nav

The split view collapses below max-width: 700sp, using native push/pop navigation and the automatic content-header back button.

Search

_on_search_changed is driven by each page's search_keywords tuples, so search coverage belongs in page classes rather than in sidebar labels.

Toast path

SettingsStore.set_toast_callback points to _show_toast; close handling calls flush_and_detach so no delayed apply targets a disposed overlay.

Cleanup

BasePage.setup_cleanup connects unrealize to unsubscribe_all; do not move this back to transient unmap cleanup.

Styling Boundary

Stock first

Most settings UI should be native Adw.PreferencesPage, Adw.PreferencesGroup, Adw.SwitchRow, Adw.ComboRow, Adw.EntryRow, and Adw.ActionRow. Do not add CSS for standard row spacing, headers, toasts, or dialogs.

Wallpaper CSS

files/usr/lib/universal-lite/settings/css/style.css owns wallpaper-grid, wallpaper-tile, wallpaper-add, wallpaper-remove, and wallpaper-placeholder.

Accent CSS

accent-swatch-grid and accent-swatch provide only the circular picker chrome; per-accent fill classes are generated at app startup from palette data.

Search focus

settings-search-entry is a narrow exception for focus ring and selection colors on the global Gtk.SearchEntry.

Safe-Edit Notes

Read the Adwaita reference

Before changing page structure, scan docs/superpowers/specs/2026-04-20-settings-adwaita-reference.md for row patterns, navigation push guidance, and dialog expectations.

Event-bus cleanup

Pages subscribe through BasePage.subscribe; outer widgets must call BasePage.setup_cleanup so unsubscribe_all runs when destroyed.

No unmap regression

Do not use unmap for event-bus cleanup. Collapsed split navigation, Adw.NavigationView pushes, and minimize can unmap live pages.

Navigation sub-pages

For Panel, Keyboard, and Network, attach cleanup to the outer Adw.NavigationView, not the root preferences page that unmaps during pushes.

Adw row patterns

Prefer Adw.SwitchRow, Adw.ComboRow, Adw.EntryRow, Adw.PasswordEntryRow, and Adw.ActionRow before inventing Gtk box layouts.

Dialogs

Use Adw.AlertDialog for confirm/revert/destructive flows. Existing Display and Network code show timer, close-response, and destructive response patterns.
