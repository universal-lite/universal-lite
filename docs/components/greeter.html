Greeter

Contributor map for GreeterWindow and GreetdClient in files/usr/bin/universal-lite-greeter. Use this page to trace the pre-login GTK kiosk flow, session chooser, palette loading, last-user state, and greetd IPC order before editing the login screen.

HTML is used here because greeter changes cross visual CSS, pre-login file state, and a strict IPC state machine; dense maps make those boundaries visible at once.

Architecture

Launch contract

main exits unless GREETD_SOCK is present, matching the greetd-launched cage kiosk assumption rather than a normal desktop app launch.

Pre-login theme

_load_accent_name, _load_palettes, _load_theme, and _build_css resolve shared palette data before GreeterApp installs the GTK provider.

Window shell

GreeterWindow fullscreens an 800x600-centered layout with clock/date above a single login card.

Auth boundary

GreetdClient exposes only create_session, post_auth_response, start_session, and cancel_session; UI state reacts to those response types.

Greetd State Map

Last User

_read_last_user checks /var/lib/greetd/last-user then /var/lib/universal-lite/last-user. If found, _show_password_state immediately pre-creates a greetd session.

Username Input

With no current user, _on_submit trims self._username_entry. Empty input shows Enter a username.; valid input advances to password state.

Create Session

_create_greetd_session sends GreetdClient.create_session. A secret auth message leaves the password field visible; info is acknowledged with post_auth_response(None).

Password Input

Password state requires non-empty self._password_entry, disables login-button, then posts the password with GreetdClient.post_auth_response.

Responses and Errors

_create_greetd_session may call _start_session directly when greetd returns success before password submission. _write_last_user runs only in password-submit success paths. auth_error becomes Incorrect password.; other errors use greetd descriptions or localized fallbacks. _reset_auth cancels, clears the password, and recreates the session for retry.

Session Start

_start_session sends the selected desktop command from _sessions. On greetd success, the GTK application quits because greetd owns the launched session.

Behavior Checklist

Last-user loading

Known users skip the username field and show Welcome back, {username}; stale greetd user errors fall back to username state and show the description.

Theme and accent files

GREETER_THEME_PATH and GREETER_ACCENT_PATH are pre-login /var/lib slots propagated by settings/session behavior, not user home files.

Session discovery

_discover_sessions parses /usr/share/wayland-sessions/*.desktop, splits Exec, and falls back to labwc when no sessions are usable.

Username/password entry

Both fields activate on Enter. Username is stripped before session creation; password is posted as typed and never written to last-user state.

Error states

_show_error writes error-label. Key presses hide the label. Connection failures re-enable submit when they occur after password submission.

Login submission

After password submission succeeds, _write_last_user stores only the username in /var/lib/greetd/last-user, then _start_session launches the selected command.

User switching

switch-user appears in password state to return to username entry. It cancels active greetd auth first, preventing a stale PAM conversation from receiving another user's password.

Time/date refresh

_update_clock sets 12-hour clock and full date text during construction and through GLib.timeout_add_seconds(30, ...).

Safe Edit Rules

greetd IPC ordering

Keep create_session before any password response, acknowledge info messages with post_auth_response(None), and call cancel_session before changing users or retrying auth.

Password handling

Do not persist, log, trim, or echo password text. Only pass self._password_entry.get_text() to post_auth_response, then clear it on reset.

Pre-login palette fallback

The greeter must render without user home access. Preserve blue, light, and hardcoded palette fallbacks for missing or invalid files.

Translation domain sharing

The file binds universal-lite-settings; new greeter strings share settings translations and belong in that domain, not a separate greeter-only domain.

Kiosk/display assumptions

The window calls fullscreen and centers a compact card for cage. Test layout on small displays before changing card width, clock size, or dropdown visibility.

Session command trust

_discover_sessions reads system desktop files and passes selected Exec tokens to greetd; avoid adding shell interpretation around session commands.
