Architecture

Window Shell

AppMenu is a decorated-off Gtk.Window manually added to the Gtk.Application. It prefers Gtk4LayerShell as an overlay layer surface and falls back to a normal toplevel when layer shell is unavailable or init fails.

Surface Placement

_setup_surface anchors the layer window to all four edges with exclusive zone 0; the visible app-menu-surface is positioned inside a full-window Gtk.Overlay by panel edge and section.

Launcher Contract

_toggle_or_lock provides click-to-toggle singleton behavior, while click-outside dismissal is handled by a capture-phase gesture comparing pointer coordinates to the surface bounds.

Behavior and State Map

PID Toggle

_toggle_or_lock writes universal-lite-app-menu.pid. A second verified instance sends SIGTERM and exits; stale or foreign locks are replaced.

VM Renderer Guard

_detect_virtualized_session runs before the GTK import and sets GSK_RENDERER=gl by default for virtualized sessions.

App Metadata Loading

_populate collects visible Gio.AppInfo entries, wraps them as AppItem, sorts by sort_key, and splices the list store.

Frequent Apps

_load_launches reads launch history; _populate_frequent shows up to one row of installed desktop ids ranked by count and recency.

Search/Filter/Category State

_category_filter starts as All Apps. _on_filter_changed selects from CATEGORY_FILTER_LABELS; _on_search_changed refilters on every query. _app_item_matches requires category match plus every lowercase search token appearing in AppItem.search_text.

Frequent Visibility

_update_frequent_visibility hides the frequent row while searching or category filtering, and also when metrics disabled frequent apps.

Grid Activation

Single-click Gtk.GridView activation and search-entry activation both call _launch_item, which launches the app, records desktop-id history, and closes the menu.

Power Confirmation

Lock runs immediately through _run_command. Logout, restart, and shutdown switch the stack to confirm; Escape cancels confirm before closing the menu.

Sizing Modes

popover keeps the ideal ChromeOS-like menu; compact removes frequent apps and shrinks margins; ultra also hides filter, section labels, and power.

Safe Edit Rules

Layer-Shell Preload

Do not move GTK imports above the LD_PRELOAD re-exec block. Gtk4LayerShell must load before libwayland-client or init can silently become a toplevel.

Low-Resolution Metrics

Keep _menu_metrics feature flags, logical display scaling, and grid height calculations together; low-resolution tests cover compact and ultra behavior.

Model/Filter Separation

Preserve the Gio.ListStore plus Gtk.FilterListModel separation. Search and category state should invalidate _grid_filter, not mutate the source app list.

Accessibility Names

Tiles and controls use _set_accessible_label with visible labels/tooltips as fallback. New controls should keep action-specific names such as Launch {app}.

Twilight Palette

_shell_theme_class intentionally selects the opposite menu palette when panel_twilight is enabled; _build_twilight_css must remain generated from palette.json values.
