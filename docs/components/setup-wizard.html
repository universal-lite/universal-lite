Page Flow Strip

Page 0_build_language_pageLanguage list, locale variants, keyboard-toggle, and keyboard-panel. Starts as the first visible page.
Page 1_build_network_pageWired status, Wi-Fi scan rows, hidden SSID form, and network status. Auto-skipped while _network_skipped stays true.
Page 2_build_disk_pageDrive, filesystem, zram/zswap choice, optional disk swap size, and destructive-drive warning.
Page 3_build_account_pageFull name, username, password, sudo checkbox, root password, and username auto-fill.
Page 4_build_system_pageTimezone detection/search and optional hostname. Memory controls live on disk page, not here.
Page 5_build_apps_pageDefault Flatpak choices from DEFAULT_FLATPAKS, displayed as app rows with checkboxes.
Page 6_build_confirm_pageSummary rows plus mandatory erase acknowledgment before Install can proceed.
Page 7_build_progress_pageProgress steps, details-toggle, log-view, retry/skip/back/reboot actions.

State and Data Map

Install Config

_step_finalize writes /var/lib/universal-lite/install-config.json with memory_strategy, swap_size_gb, and username via _atomic_write_text mode 0o600.

Flatpak Selection

_get_selected_apps feeds _step_save_app_selection. _flatpak_app_list_contents writes a deterministic newline list or zero-byte file; pinned defaults follow _pinned_apps_for_selected_flatpaks.

Disk and Filesystem

_setup_target_drive and _setup_filesystem drive _step_partition_disk, which wipes, partitions, formats ext4/xfs/btrfs, mounts root, boot, and EFI.

Memory Strategy

_get_swap_config selects zram only or zswap with 2/4/8/custom GB. _step_configure_memory writes zram-generator config or enables the zswap/encrypted swap units.

Network State

_network_skipped, _connected_ssid, _wifi_device, and connectivity checks decide whether the network page appears and whether Confirm can install.

Validation Gates

_validate_page delegates to _validate_disk, _validate_account, _validate_system, and _validate_connectivity. Confirm also requires _confirm_ack.

Skip Behavior

Navigation skips Network when already online. Progress supports skippable step behavior through _progress_skip, though current install steps are fatal or retry.

Async Install Execution

_start_step_runner runs install steps in a daemon thread, updates GTK through GLib.idle_add, and toggles SELinux permissive while post-install files are written.

Atomic Writes

_atomic_write_text protects state files with tmp/fsync/rename. _sysroot_unlink_write breaks ostree hardlinks before atomic sysroot writes.

Safe Edit Rules

Translation Markers

Visible strings use _(); deferred labels use N_(). New dynamic text should register in self._retranslatable or be refreshed by _retranslate.

Low-Memory Safeguards

Do not remove the log buffer cap, tail deque, temp-log-on-target behavior, or 256-character password limits; they protect 2 GB devices.

Subprocess Behavior

Long commands run through _run_logged or controlled subprocess.Popen with start_new_session, timeouts, and child signal reset.

Password Limits

Gtk.PasswordEntry uses activate signals instead of unavailable set_activates_default; _limit_entry_length and validation cap passwords at 256 chars.

Alert Dialog Compatibility

_show_install_in_progress_alert guards Gtk.AlertDialog with hasattr and falls back to a modal Gtk.Window; do not reintroduce Gtk.MessageDialog.

Accessibility Announcements

Labels bind to entries through _bind_label_to_entry. Shared bottom status updates call _announce_status via _set_status; network-page status labels may update directly and intentionally avoid removed live-region APIs.
