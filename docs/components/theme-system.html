Theme System

Universal-Lite themes are data-first: palette.json defines light, dark, and accent values; runtime helpers apply those values to shell surfaces, persisted greeter state, and component CSS. This page maps that path across settings, app-menu, greeter, apply-settings, and twilight panel behavior.

State Map: Palette To Surfaces

Shared data

palette.json owns light, dark, and accents; FALLBACK_PALETTE in theme.py mirrors safe defaults.

->

App-menu GTK tokens

gtk_color_defines loads settings, normalizes theme, accent, and high_contrast, then emits @define-color CSS for app-menu.

->

Shell surfaces

Generated tokens feed app-menu BASE_CSS; settings style.css consumes Adwaita/GTK tokens and accent CSS; greeter builds direct CSS from loaded palettes.

Accent swatches

_build_accent_css reads PALETTE_PATH, emits .accent-<name> rules, and refreshes through _reload_accent_css.

->

Twilight override

_build_twilight_css generates app-menu-shell-dark and app-menu-shell-light palettes so menu tint matches panel twilight.

->

Pre-login mirror

universal-lite-apply-settings writes /var/lib/universal-lite/greeter-theme and /var/lib/universal-lite/greeter-accent; greeter reads those paths before login.

Ordering matters: app-menu CSS expects generated @accent_color, @accent_fg_color, @window_bg_color, @window_fg_color, and @borders to exist before its token-consuming rules load; settings loads base Adwaita/GTK token consumers plus generated accent swatch rules.

Token And Data Map

gtk_color_defines

Returns one GTK CSS string of @define-color declarations for app-menu, selecting palette colors through normalize_theme, normalize_accent, and accent_foreground.

_build_accent_css

Settings app runtime CSS: sets active accent_color, accent_fg_color, then generates swatch fill and checked-icon contrast rules from each accent.

_build_twilight_css

App-menu runtime CSS: converts light and dark palette entries to translucent shell tokens for app-menu-shell-* classes.

DARK_THEME_COLORS

Setup-wizard-local dark CSS map used by files/usr/bin/universal-lite-setup-wizard; do not treat it as the source of current shared palette truth.

@accent_color / @accent_fg_color

Accent fill and readable foreground pair used by focus, selection, wallpaper checks, and active accent states; destructive power affordances use error/destructive colors instead.

@window_bg_color / @window_fg_color / @borders

Core surface, foreground, and edge tokens consumed by settings CSS and app-menu CSS; high_contrast forces black, white, and white border definitions.

Behavior Matrix

Light / Dark Palette

palette.json provides window_bg, view_bg, headerbar_bg, card_bg, fg, secondary_fg, border, and inactive_fg for both modes.

Accent Selection

normalize_accent falls back to blue; settings swatches use generated .accent-blue through .accent-slate classes from palette data.

High Contrast

high_contrast in theme.py normalizes theme to dark and emits black background, white foreground, and white borders for GTK token consumers.

Panel Twilight

universal-lite-apply-settings applies panel_twilight by drawing panel tokens from the opposite palette; app-menu mirrors this with _shell_theme_class returning the inverse shell class.

Greeter Propagation

universal-lite-apply-settings owns propagation to /var/lib/universal-lite/greeter-theme and /var/lib/universal-lite/greeter-accent; greeter reads them and calls _build_css(PALETTES[theme]).

App-menu Inversion

app-menu-shell-dark is used over a light user theme when twilight is on; app-menu-shell-light is used over a dark user theme.

Settings CSS Tokens

style.css consumes Adwaita/GTK @window_fg_color, @window_bg_color, @accent_color, @accent_fg_color, and @error_color; _build_accent_css generates active @define-color accent_color / @define-color accent_fg_color values plus per-accent swatch rules.

Fallback Paths

load_palette tries supplied paths before FALLBACK_PALETTE; app-menu also checks a repo palette path before disabling twilight CSS.

Safe Edit Rules

Preserve Contrast

Run accent changes through the existing contrast helpers: accent_foreground, _accent_check_fg, and greeter _accent_foreground choose readable foregrounds.

Keep CSS Order

Generated @define-color rules must stay before BASE_CSS, settings swatch rules must load after base style.css, and twilight overrides must remain last.

Respect Fallbacks

When editing palette lookup, preserve installed and repo fallback paths: DEFAULT_PALETTE_PATH, _REPO_PALETTE_PATH, and greeter hardcoded fallback.

Avoid Duplicate Hex

Do not add hardcoded duplicates of palette colors where @accent_color, @window_bg_color, @window_fg_color, @borders, or generated swatches already exist.
